## 2. プロジェクト原則

1. **「企業」を標準型にしない** — 家・宗族・修道院・ワクフ・ギルド・DAO を同等に比較
2. **すべての記述を `claim` として扱う** — 出典・確度・解釈者・記録時点を必須化
3. **生命メタファーは限定的比喩として使う** — 観察次元を増やすために使い、説明装置にしない
4. **生存を適応の証拠にしない** — 適応度・正統性・資源を結果と独立に測る
5. **claim 値を 5 値化** (`present | absent | partial | unknown | inapplicable`) — Seshat databank の Whitehouse 2019 *Nature* 撤回事件 (NA→Absent 変換問題) を踏まえた厳格化

## 3. 採用した理論的立場

| 系譜 | 主要人物 | 採用 | 理由 |
|---|---|---|---|
| 個体群生態学 | Hannan & Freeman (1977/89) | **主理論** | 多数組織の比較・存続淘汰・環境適合の操作化 |
| オートポイエーシス | Maturana-Varela / Luhmann | 補助 | 境界・決定・意味の分析 |
| サイバネティクス | Stafford Beer (1972-) | 補助 | 分権/統合・現業/方針・情報遅延の診断 |
| 制度論・資源依存論 | DiMaggio-Powell / Pfeffer-Salancik | 補正項 | 「適応の物語」を正統性・模倣・強制で補正 |
| 有機体的社会観 | Herbert Spencer (19c) | **批判対象** | 単線進歩史観の警戒標識 |

### 生命メタファーの 5 罠と対処

1. **Reification** — 「組織が呼吸する」のような表現を避け、観察可能な作動を毎回明示
2. **適者生存の循環論法** — 適応度を結果と別に measurable 変数として記録
3. **個体境界の曖昧さ** — 法的境界・成員境界・資産境界・意思決定境界・通信境界を分けて記録
4. **機能主義の後知恵** — 効率以外の説明候補 (権威・正統性・偶然・模倣) を併記
5. **生物学からの過剰輸入** — 機能アナロジー (淘汰・代謝・繁殖) を概念ごとに対応関係を限定

| **VSM S5 Policy/Identity** | **13** | 最頻出。組織の存続が「方針・同一性」維持と密接 |
| Miller M02 Boundary | 11 | 境界維持は普遍的だが、引き方は多様 (法・血縁・コード) |
| Miller M17 Memory | 10 | 記憶機構 (写本・帳簿・ブロックチェーン) も時代横断的 |

### Miller vs VSM 比

- Miller Living Systems: 53 records / 25 機能
- Beer VSM: 24 records / 5 機能
- VSM は records 数は少ないが S5 で最頻出 → identity / normative closure は時代を超えて annotate しやすい

### 拡張提案 (codex 7 → codex 7 extension)

25 マスター機能を維持しつつ、デジタル組織で前景化する **70 sub-function** を提案:
- M14 Timer: Ethereum block time, debt auction epoch, Cistercian General Chapter (年 1)
- M11 Input Transducer: oracle signal, sensor, API endpoint, petition
- M20 Output Transducer: on-chain event, webhook, leaked document

未使用 10 機能の解釈導入で **35-40 件の新規 function_record** が可能と試算。

## 7. 主要な経験的発見 10 件

1. **大成分の 1500 年連続性**: ベネディクト会 (6c) → Cluny → Cistercians → Bologna → ハンザ → VOC → MakerDAO の単一連結成分が 1500 年存続。修道会の継承メカニズムが世俗組織にミメティックに伝播。

2. **VSM S5 (Policy/Identity) の最頻出性**: 43 ケースのうち 13 で記録。組織の長寿命は「方針・同一性」の維持と密接。

3. **東西分岐**: 東アジア成分 (比叡山 + 三井 + 鴻池 + 京セラ + 山西商幫 + 堺会合衆 + 安東権氏門中) は西洋成分から制度的に分岐。Greif の制度的経路依存性と一致。

4. **collegia → ボローニャ大学の 2,000 年制度継承**: Justinian Digest の corpus/collegium 概念が中世 universitas 法人理論の基礎。古代-中世-現代の連続線。

5. **Sakai = "Japanese Hansa"**: 16 世紀の堺会合衆が parallel evolution として中世ハンザ自治都市と機能的同型。

6. **「公私の境界を縫う形態」が現代組織の共通パターン**: ICRC (Swiss Verein + 国際法委任) / UN (国家ベース機関 + 私的 NGO 准国家承認) / DARPA (連邦機関 + adhocracy) / Wikimedia (501c3 + community) / MakerDAO (Foundation 解体 + DAO)。

7. **三井のみが「reorganization → dissolution → revival」三段階を経験**: 1909 三井合名 → 1947 GHQ 解体 → 1955 keiretsu 再編。selection を経て retention に転じた唯一のケース。

8. **SEWA Bank (1974) が Grameen (1983) に 9 年先行**: マイクロファイナンス史の修正。貯蓄主導型 vs 信用主導型の対照を確立。

9. **timar → Süleymaniye の supply_chain 制度的接続**: miri (国有地) が皇室 mülk 化を経由して waqf へ再変換され、軍事徴税ネットワークが文化福祉複合体へ資産転送。

10. **claim 値 5 値化の有用性**: Hadza の `start_date='inapplicable'` のように、狩猟採集バンドには「創立日」概念が適用できないことを明示的に記録できる。Seshat 撤回事件の防止策として機能。

## 10. 既知の制約と限界

1. **PostgreSQL 環境未構築**: SQLite サンドボックスで開発中。本番移行は Phase 4-5 で実施 (`plan/postgres_migration.md` 参照)。

2. **Wikidata 取り込みの品質問題**: P31/P279* (subclass tree) が広く展開し、historical_organization 配下に医療センター・フェリー等が混入。`etl/17_dedupe_quality.py` で 76 件信頼性スコア再算出済。

3. **古代カバレッジの薄さ**: BC3500 ウルク神殿が最古だが、メソアメリカ・南アジア・サブサハラ古代国家の事例は未注釈。

4. **個体境界の曖昧さ**: 三井 (家業 → 法人 → 系列)、MakerDAO (Foundation → DAO)、Wikimedia (財団 + コミュニティ) について、法的境界と意思決定境界のずれを感度分析していない。

5. **サンプルサイズ 18-43 の限界**: クラスタリング (codex 13) は 18 ケースで実施。統計的検出力は限定的。

6. **西洋・近代法人バイアス**: 完全注釈 43 ケースのうち 27 が西洋 (12 中世 + 8 近世西洋 + 7 19-20c 西洋)、東アジア 9、イスラム 5、その他 2。Phase 4-5 で東南アジア・サブサハラ・先住民組織を補強予定。

7. **生命メタファー罠の検証**: codex 20 が 5 罠の自動検出ハーネスを実装したが、現状の検出は表面的 (語彙ベース)。意味論的検証 (LLM ベース) は Phase 6 で実施。
